之前降低了projection两个模态的维度，但是没对低维的infoNCE上添加防止坍缩的约束。加上也没啥用
根本问题                                                                                                                                  
  这是跨模态对比学习（观察 z1 vs 动作 z2）：                                          
  - 两个独立投影层随机初始化                                                          
  - 投影到的 256d 空间本来就不对齐                                                    
  - InfoNCE 假设"正样本应该相似"，但两个空间根本没有共同基础

InfoNCE 为什么不能对齐？InfoNCE 优化的是 softmax 概率分布：它只关心 相对大小，不关心绝对值
pos 和 neg 一起涨/跌时，softmax 比例不变，loss 不变，没有梯度

MSE 是 绝对约束：直接要求正样本对在 256d 空间中的 位置重合，强迫两个投影层学习 
兼容的映射：对于匹配的 (观察, 动作) 对，输出相同的 256d 向量

总结原因：之前方案失败的根本原因                                               
                                                                                 
  1. 当前架构的信息流                                                            
                                                                                 
  观察侧 (z1):                                                                   
  VLM输出 → cmp_projection → [batch, 1024] → proj_z1 → [batch, 1024] →           
  infonce_proj_z1 → [batch, 256]                                                 
                                                                                 
               ↓                                                                 
                                                                                 
       cosine similarity                                                         
                                                                                 
               ↑                                                                 
  动作侧 (z2):                                                                   
  动作序列 → SingleHeadContentAttention → [batch, 1024] → proj_z2 → [batch, 1024]
   → infonce_proj_z2 → [batch, 256]                                              
                                                                                 
  2. 核心失败原因：两个独立随机空间的对齐问题                                    
                                                                                 
  2.1 初始状态：两个不相干的空间                                                 
                                                                                 
  - proj_z1 和 proj_z2 是随机初始化的独立网络                                    
  - 它们把两个语义完全不同的输入（VLM视觉特征 vs                                 
  动作序列特征）映射到"同一个"1024维空间                                         
  - 但这两个映射没有任何先验关联                                                 
                                                                                 
  类比：这就像两个翻译员，一个把中文翻译成"某种语言A"，另一个把日语翻译成"某种语 
  言B"，然后期望 A 和 B 是同一种语言——这是不可能的。                             
                                                                                 
  2.2 InfoNCE 为什么不能自举对齐？                                               
                                                                                 
  InfoNCE 的数学形式：                                                           
  L = -log(exp(sim(z1[i], z2[i])/τ) / Σ_j exp(sim(z1[i], z2[j])/τ))              
                                                                                 
  关键问题：InfoNCE 是相对损失，只优化"正样本相似度 > 负样本相似度"。            
                                                                                 
  当两个空间初始不对齐时：                                                       
  - sim(z1[i], z2[i]) ≈ sim(z1[i], z2[j]) ≈ 随机值（因为投影层随机）             
  - 梯度会同时影响所有相似度，导致 pos 和 neg 一起涨或一起跌                     
  - 没有"锚点"告诉模型哪个方向是正确的                                           
                                                                                 
  实验现象：pos: 0.85, neg: 0.85, gap: 0.00                                      
                                                                                 
  2.3 MSE 为什么导致坍缩？                                                       
                                                                                 
  MSE 是绝对损失：直接最小化 ||z1[i] - z2[i]||²                                  
                                                                                 
  看似解决：强制拉近正样本对，建立对齐。                                         
                                                                                 
  实际问题：MSE 找到的"最优解"是所有向量都相同！                                 
  - 如果 z1[i] = z2[i] = c（常向量），MSE = 0                                    
  - 模型会学到 proj_z1(任何输入) → c，proj_z2(任何输入) → c                      
  - 这就是表示坍缩                                                               
                                                                                 
  实验现象：pos: 0.85 → 0.99, neg: 0.85 → 0.99（所有相似度都变成1）              
                                                                                 
  2.4 Margin Loss 为什么也不行？                                                 
                                                                                 
  Margin Loss：max(0, margin - (pos - hardest_neg))                              
                                                                                 
  问题：                                                                         
  - Margin 只约束 pos - neg > margin，但不提供对齐信号                           
  - 当两个空间完全不对齐时，模型可以通过让所有向量都相似来满足约束               
  - 或者让所有向量都很远（pos 和 neg 都是负值，满足 margin）                     
                                                                                 
  3. 失败方案总结表                                                              
  ┌────────────┬──────────────────────┬────────────────────┬──────────────┐      
  │    方案    │       数学目标       │      失败模式      │     根因     │      
  ├────────────┼──────────────────────┼────────────────────┼──────────────┤      
  │ 纯 InfoNCE │ pos > neg (相对)     │ pos ≈ neg 同涨同跌 │ 无对齐锚点   │      
  ├────────────┼──────────────────────┼────────────────────┼──────────────┤      
  │ + MSE (强) │ z1[i] = z2[i] (绝对) │ 全坍缩             │ 常数解最优   │      
  ├────────────┼──────────────────────┼────────────────────┼──────────────┤      
  │ + MSE (弱) │ 同上但弱化           │ 慢坍缩             │ 同上，只是慢 │      
  ├────────────┼──────────────────────┼────────────────────┼──────────────┤      
  │ + Margin   │ pos - neg > m        │ pos ≈ neg          │ 无对齐信号   │      
  ├────────────┼──────────────────────┼────────────────────┼──────────────┤      
  │ 共享投影   │ 减少参数             │ 仍坍缩             │ 仍无交互     │      
  └────────────┴──────────────────────┴────────────────────┴──────────────┘      
  4. 根本矛盾                                                                    
                                                                                 
  问题本质：我们希望学习一个跨模态对应关系，但：                                 
                                                                                 
  1. InfoNCE 需要已有的对应关系才能优化（鸡生蛋）                                
  2. MSE 可以建立对应关系，但会导致坍缩（杀死鸡）                                
  3. 防坍缩损失只能减缓坍缩，不能解决对齐问题                                    
                                                                                 
  ---                                                                            
  新方案思路：Cross-Attention 建立内在关联                                       
                                                                                 
  核心洞察                                                                       
                                                                                 
  之前的方案都是：先独立处理两个模态，再比较。                                   
                                                                                 
  问题是"比较"的基础不存在。                                                     
                                                                                 
  新思路：先让两个模态交互，再比较。                                             
                                                                                 
  之前（失败）：                                                                 
  z1 → proj_z1 → z1_proj ─┐                                                      
                          ├─ compare（无共同基础）                               
  z2 → proj_z2 → z2_proj ─┘                                                      
                                                                                 
  新方案：                                                                       
  z1 ─────────────────────┐                                                      
                          ├─ CrossAttn(Q=z1, K=z2, V=z2) → z1_attended           
  z2 ─────────────────────┘                                                      
                          ↓                                                      
  z1_attended 与 z2 有了内在关联 → 再做对比学习                                  
                                                                                 
  为什么 Cross-Attention 能解决问题？                                            
                                                                                 
  1. 建立关联的机制：                                                            
    - z1_attended[i] = Σ_j softmax(z1[i] · z2[j] / √d) * z2[j]                   
    - z1_attended 是 z1 对 z2 的加权表示                                         
    - 正样本对：z1[i] 和 z2[i] 来自同一个 episode，应该有高 attention            
  2. 自然的对齐信号：                                                            
    - Cross-Attention 本身就在学习"哪个 z2 与当前 z1 最相关"                     
    - 这是对比学习需要的先验信息                                                 
  3. 避免坍缩：                                                                  
    - 不是强制 z1 = z2，而是让 z1 "看到" z2                                      
    - 结果 z1_attended 仍然保留 z1 的信息（通过残差连接）                        
    - 但获得了与 z2 对齐的能力

Section 9: ForwardModel — 嵌入空间前向动力学

=== 问题背景 ===
专家数据没有漂移场景 → 没有负样本 → ITM 无法训练漂移检测能力。
Eval 时 ITM 分数在成功/失败 episode 中无显著差异，验证了这一判断。

=== 核心思路 ===
"在状态 z1 下执行动作 z2，预期到达状态 predicted_z3" —— 嵌入空间中的前向动力学模型。
ForwardModel 是回归任务，不需要负样本，只需专家的 (obs_t, action, obs_{t+k}) 三元组。
漂移信号在 eval 时自然产生：actual_z3 偏离 predicted_z3 → drift。

=== 代码改动（modeling_pi05.py） ===

1. ForwardModel 类 (line ~683)
   残差 MLP: predicted_z3 = z1 + MLP(concat(z1, z2))
   MLP 只学 delta，z1 作为 baseline，连续观察变化小更容易学。

2. PI05Pytorch.__init__ (line ~1831)
   self.forward_model = ForwardModel(hidden_dim=512)

3. forward_cmp 中 L_forward (line ~2230)
   if cmp_vec_2 is not None:
       predicted_z3 = self.forward_model(z1, z2)
       l_forward = MSE(predicted_z3, z3.detach())
       loss_scalar += l_forward
   z3.detach()：不让 L_forward 梯度回传到 ObsQueryBridge 的 z3 编码路径。
   日志字段：cmp/l_forward, cmp/forward_l2

4. reset() 中 drift 跟踪字段 (line ~2814)
   _predicted_z3, _drift_scores, _drift_chunk_ids, _drift_segment_ids

5. _encode_obs 辅助方法 (line ~3038)
   提取 VLM 编码逻辑为独立 @torch.no_grad() 方法，避免重复代码。

6. _should_replan 重构 (line ~3063)
   - 接收可选 z1 参数（已预计算时跳过 VLM 编码）
   - 内部调用 _encode_obs
   - 返回 3-tuple: (should_replan, itm_score, z1)

7. predict_action_chunk (line ~3176)
   生成 chunk 后，用 ForwardModel(z1, z2) 预测首段 predicted_z3 并存储。

   - 为下一段用 ForwardModel(actual_z3, z2_next) 生成新的 predicted_z3

=== Eval 时 drift 检测流程 ===
chunk 生成(obs_t) → z1, z2 → predicted_z3 = FM(z1, z2)
执行 10 步 → obs_{t+10} → actual_z3 = encode(obs_{t+10})
drift = ||predicted_z3 - actual_z3||
drift < 阈值 → 继续用旧 chunk segment
drift > 阈值 → replan

=== 关于 ITM 负样本的讨论 ===
当前 ITM 负样本是 batch 内 cross-pair（别的 sample 的 action），语义上不正确：
- 同任务不同 episode：obs 相似，action 也相似，被当负样本不合理
- 不同任务：区分太简单，学不到有意义的东西
- 真正的负样本应该是"当前 obs 下不合适的动作"

考虑的改进方向：同 episode 时序错位构造负样本
- 正样本：(obs_t, model_action[0:att_len]) — z2 来自模型生成（设计意图不变）
- 负样本：(obs_t, GT_action[offset:offset+att_len]) — GT 时序错位
  语义：obs_t 不该执行 30 步以后才该做的动作




当前 Loss 完整实现总结（含 L2 方案 A）                                                                     
                                                                                                             
  ---                                                                                                        
  顶层结构                                                                                                   
                                                                                                             
  Total Loss = bc_loss + 0.1 × cmp_loss                                                                      
                                                                                                             
  - bc_loss：MSE，VLA 全 18 层生成动作 vs GT action                                                          
  - cmp_loss：对比匹配预测，包含 4 个子项                                                                    
                                                                                                             
  ---                                                                                                        
  CMP Loss 详细组成                                                                                          
                                                                                                             
  cmp_loss = cross_loss + structure_loss + anticollapse_loss + l_forward                                     
           = (L1 + L2) + structure + (variance + covariance) + l_forward                                     
                                                                                                             
  ---                                                                                                        
  1. 跨模态匹配 — Cross Loss（ITM）                                                                          
                                                                                                             
  基于 ALBEF 的 ITM Head：concat(z1, z2, z1-z2, z1*z2) → MLP → 二分类 logit                                  
                                                                                                             
  L1：obs_t ↔ action_t（L1 改进后）                                                                          
  Column 1: 正样本                                                                                           
  来源: (z1, z2) — obs_t 配模型生成的 action[0:att_len]                                                      
  标签: 1                                                                                                    
  数据流: z2 = model_action → ContentAttn → proj_z2                                                          
  ────────────────────────────────────────                                                                   
  Column 1: 负样本                                                                                           
  来源: (z1, z2_neg) — obs_t 配同 episode 跨 chunk GT 时序错位 action                                        
  标签: 0                                                                                                    
  数据流: z2_neg = GT_action[offset:offset+att_len] → 同路径编码                                             
  - offset 范围：[current_pos + chunk_size, ep_len - att_len]，随机，至少跨 chunk                            
  - validity：episode 太短无法跨 chunk 时标记 invalid，不参与 loss                                           
  - loss：(BCE(pos,1) + masked_BCE(neg,0)) / 2                                                               
  - 语义：obs_t 不该执行 30+ 步以后才该做的动作                                                              
                                                                                                             
  L2：action_t ↔ obs_{t+k}（L2 改进后 — 方案 A）                                                             
  Column 1: 正样本                                                                                           
  来源: (z2, z3) — action_t 配未来 obs_{t+k}                                                                 
  标签: 1                                                                                                    
  数据流: z3 = obs_{t+k} → VLM 前 6 层 → ObsQueryBridge                                                      
  ────────────────────────────────────────                                                                   
  Column 1: 负样本                                                                                           
  来源: (z2, z3_neg) — action_t 配同 episode 远离 t+k 的 GT obs                                              
  标签: 0                                                                                                    
  数据流: z3_neg = obs_{t+m} → 同路径编码（额外 VLM forward）                                                
  - offset 范围：[0, k - chunk_size] ∪ [k + chunk_size, ep_len - 1]，随机，确保 |m - k| >= chunk_size        
  - validity：episode 太短无法采样远离 k 的时刻时标记 invalid                                                
  - 成本：额外一次 VLM 前 6 层 forward，训练时间 +19%，显存 +25-30%                                          
  - loss：(BCE(pos,1) + masked_BCE(neg,0)) / 2                                                               
  - 语义：执行 action_t 后到达 obs_{t+k}，不是远处的 obs_{t+m}                                               
                                                                                                             
  ---                                                                                                        
  2. 结构保持 — Structure Loss                                                                               
                                                                                                             
  保持 z2 嵌入空间中 top-k 邻居的排序和原始 action 空间一致：                                                
                                                                                                             
  # 取 action 空间 L2 距离矩阵的 top-k 最近邻                                                                
  orig_knn_dists = action_distances[i, top_k_indices]                                                        
  # 计算 z2 空间对应的 learned distance                                                                      
  learned_knn_dists = z2_distances[i, top_k_indices]                                                         
  # Soft Rank Loss + Pair-order Loss                                                                         
  loss = rank_loss + 0.1 × pair_loss                                                                         
                                                                                                             
  功能：防止对比学习破坏 action 之间的相对距离关系                                                           
                                                                                                             
  ---                                                                                                        
  3. 防坍缩 — Anti-collapse Loss（VICReg 风格）                                                              
                                                                                                             
  # Variance Loss: 各维度标准差不低于 γ=0.4                                                                  
  variance_loss = mean(relu(γ - std(z1))) + mean(relu(γ - std(z2)))                                          
                                                                                                             
  # Covariance Loss: 协方差矩阵非对角元素趋零                                                                
  cov_z1 = (z1_centered.T @ z1_centered) / (B - 1)                                                           
  covariance_loss = sum(cov_z1[off_diag]²) / dim²                                                            
                                                                                                             
  功能：                                                                                                     
  - Variance：防止维度坍缩（所有样本在某维度上值相同）                                                       
  - Covariance：促进维度去相关（不同维度学习独立特征）                                                       
                                                                                                             
  ---                                                                                                        
  4. 前向动力学 — L_forward（ForwardModel）                                                                  
                                                                                                             
  predicted_z3 = z1 + MLP(concat(z1, z2))   # 残差结构                                                       
  l_forward = MSE(predicted_z3, z3.detach())                                                                 
                                                                                                             
  训练功能：学习 (obs_t, action_t) → obs_{t+k} 的嵌入空间映射                                                
                                                                                                             
  Eval 功能：                                                                                                
  drift = ||predicted_z3 - actual_z3||₂                                                                      
  if drift > threshold:                                                                                      
      trigger_replan()  # 检测执行偏移，触发重规划                                                           
                                                                                                             
  ---                                                                                                        
  数据编码路径                                                                                               
  ┌─────────────────┬──────────────────────────────────────────────────────────┬──────────┐                  
  │      特征       │                         编码路径                         │   维度   │                  
  ├─────────────────┼──────────────────────────────────────────────────────────┼──────────┤                  
  │ z1 (obs_t)      │ obs → VLM 前 6 层 → ObsQueryBridge (64 queries)          │ [B, 512] │                  
  ├─────────────────┼──────────────────────────────────────────────────────────┼──────────┤                  
  │ z2 (action_t)   │ model_action[0:10] → ContentAttention → proj_z2          │ [B, 512] │                  
  ├─────────────────┼──────────────────────────────────────────────────────────┼──────────┤                  
  │ z3 (obs_{t+k})  │ future_obs → VLM 前 6 层 → ObsQueryBridge                │ [B, 512] │                  
  ├─────────────────┼──────────────────────────────────────────────────────────┼──────────┤                  
  │ z2_neg (L1 neg) │ GT_action[offset:offset+10] → ContentAttention → proj_z2 │ [B, 512] │                  
  ├─────────────────┼──────────────────────────────────────────────────────────┼──────────┤                  
  │ z3_neg (L2 neg) │ obs_{t+m} → VLM 前 6 层 → ObsQueryBridge                 │ [B, 512] │                  
  └─────────────────┴──────────────────────────────────────────────────────────┴──────────┘

  梯度回传范围                                                                                               
                                                                                                             
  bc_loss 梯度:

    ├─ Expert 全 18 层                                                                                       
    ├─ action_in/out_proj                                                                                    
    └─ time_mlp                                                                                              
                                                                                                             
  cmp_loss 梯度:                                                                                             
    ├─ VLM 前 6 层 (forward_partial)                                                                         
    ├─ cls_head_prefix                                                                                       
    ├─ ObsQueryBridge (z1, z3, z3_neg 编码器)                                                                
    ├─ ContentAttention (z2, z2_neg 编码器)                                                                  
    ├─ proj_z2                                                                                               
    ├─ ITMHead

  ├──────────────┼────────────────────────────┼────────────────────┼───────────────────────────────┤

  当前 Pipeline 总结                                                                       
                                                                                           
  编码器                                                                                   
                                                                                           
                          ┌─ obs_query_bridge ─→ z1 [B,512]  (obs_t)                       
  obs 图像 → VLM 前6层 ──┤                                                                 
                          └─ obs_query_bridge ─→ z3 [B,512]  (obs_{t+k})                   
                                                 z3_neg [B,512] (obs_far)                  
                                                                                           
  action → ContentAttention → proj_z2 ─→ z2 [B,512]  (action_t)                            
                                          z2_neg [B,512] (action_neg)                      
                                                                                           
  z1/z3/z3_neg 共享同一个 VLM + obs_query_bridge；z2/z2_neg 共享 ContentAttention +        
  proj_z2。                                                                                
                                                                                           
  Loss 组成                                                                                
                                                                                           
  当前模式 cmp_pretrain=False，总 loss：                                                   
                                                                                           
  total = bc_loss + 0.1 × cmp_loss                                                         
                                                                                           
  其中 cmp_loss 内部：                                                                     
                                                                                           
  cmp_loss = cross_modal + structure + anticollapse + l_forward                            
  Loss: L_cross_L1                                                                         
  公式: ITMHead(z1, z2) pos/neg BCE                                                        
  梯度流向: → ITMHead, obs_query_bridge, ContentAttention, proj_z2, VLM                    
  作用: z1↔z2 跨模态匹配                                                                   
  ────────────────────────────────────────                                                 
  Loss: L_cross_L2                                                                         
  公式: ITMHead(z2, z3) pos/neg BCE                                                        
  梯度流向: → ITMHead, obs_query_bridge, ContentAttention, proj_z2, VLM                    
  作用: z2↔z3 因果链匹配                                                                   
  ────────────────────────────────────────                                                 
  Loss: L_structure                                                                        
  公式: soft rank consistency on z2 neighbors                                              
  梯度流向: → ContentAttention, proj_z2                                                    
  作用: 保持 z2 空间中 action 邻居排序                                                     
  ────────────────────────────────────────                                                 
  Loss: L_anticollapse                                                                     
  公式: variance + covariance on z1, z2                                                    
  梯度流向: → obs_query_bridge, ContentAttention, proj_z2, VLM                             
  作用: 防止 z1/z2 坍缩                                                                    
  ────────────────────────────────────────                                                 
  Loss: L_forward                                                                          
  公式: 1 - cos_sim(ForwardModel(z1.detach(), z2.detach()), z3.detach())                   
  梯度流向: → 仅 ForwardModel                                                              
  作用: 前向动力学方向预测                                                                 
  梯度流详细图                                                                             
                                                                                           
  bc_loss ──────────────────────────→ 整个模型（VLM全部层 + action expert + ...）          
                                                                                           
  L_cross_L1 (z1↔z2) ──┬──→ ITMHead                                                        
                        ├──→ obs_query_bridge ──→ VLM 前6层                                
                        └──→ ContentAttention + proj_z2                                    
                                                                                           
  L_cross_L2 (z2↔z3) ──┬──→ ITMHead                                                        
                        ├──→ obs_query_bridge ──→ VLM 前6层  (通过 z3)                     
                        └──→ ContentAttention + proj_z2      (通过 z2)                     
                                                                                           
  L_structure ──────────→ ContentAttention + proj_z2  (只涉及 z2)                          
                                                                                           
  L_anticollapse ───────┬──→ obs_query_bridge ──→ VLM 前6层  (通过 z1)                     
                        └──→ ContentAttention + proj_z2      (通过 z2)                     
                                                                                           
  L_forward ────────────→ ForwardModel 仅此一个  (输入 z1,z2 全 detach, 目标 z3 也 detach) 
                                                                                           
  L_forward 的隔离设计                                                                     
                                                                                           
  predicted_z3 = ForwardModel(z1.detach(), z2.detach())                                    
  l_forward = (1 - cosine_similarity(predicted_z3, z3.detach())).mean()                    
                                                                                           
  三个 detach 保证 l_forward 的梯度只更新 ForwardModel 自身（2层                           
  MLP，~0.5M参数），不影响任何编码器。Cosine loss 使其对 norm 膨胀免疫。                   
                                                                                           
  Eval 时 ForwardModel 用途                                                                
                                                                                           
  chunk 开始: z1 = encode(obs_t), z2 = encode(action)                                      
              predicted_z3 = ForwardModel(z1, z2)                                          
                                                                                           
  每段执行后: actual_z3 = encode(actual_obs)                                               
              drift = 1 - cosine_similarity(predicted_z3, actual_z3)                       
              如果 drift 大 → 触发 replan                                                  
                                                                                           
  下一段预测: predicted_z3_next = ForwardModel(actual_z3, z2_next)                         
                                                                                           
  训练和 eval 统一使用 cosine distance。

=== 参考论文 ===
A2C2: Action-Aware Contrastive Correction (arXiv:2509.23224, Sep 2025, University of Tokyo)
核心方法：冻结 base policy，训练轻量 correction head，每步用最新 obs 修正 action chunk
公式：a_exec = a_base + Δa, 其中 Δa = π_a2c2(obs_latest, a_base, τ_k, z_base, l)
LIBERO Spatial 结果：72.2%→81.6% (e=50,d=0), 64.4%→84.2% (e=40,d=10)

=== A2C2 可借鉴改进当前 Online Training 的三个方向 ===

方向 1: 残差监督替代/补充 feature loss
  当前: feature_loss = relu(||gen_feature - pred_feature||² - 0.1) — 间接监督
  借鉴: 加入直接残差项 ||generated_action - gt_action||
  理由: A2C2 证明直接残差监督收敛更快，feature distance 作为辅助正则化保留

方向 2: Step-level 细化
  当前: online loss 在 chunk 级别计算
  借鉴: 对 chunk 中每个 action step 分别计算与实际执行效果的偏差
  实现: 利用 rollout 中逐步收集的 obs 序列，对 chunk 中的每个位置
        计算 step-wise loss，权重可按时间步衰减（近步权重大）

方向 3: Offline 数据预构造
  当前: 必须等 online rollout 数据积累才能开始 online training
  借鉴: A2C2 先对 offline dataset 跑一遍 base policy inference 得到 a_base，
        用 a_gt - a_base 构造训练目标
  实现: 在 online training 启动前，先用 base checkpoint 对 offline dataset
        做一遍 inference，收集 (obs, a_base, a_gt) 三元组，
        这样无需等 online 数据积累就能开始训练 correction 能力
  注意: 额外计算成本（需要对整个 dataset 跑 inference），但只需做一次

=== 与 CMP + ForwardModel 的关系 ===

A2C2 和 CMP 是互补而非替代关系：
- CMP 解决 "何时需要修正"（drift 检测 → replan 决策）
- A2C2 风格的 correction 解决 "如何修正"（直接调整 action）
- 当前系统的修正方式是 replan（重新跑完整 inference），代价较大
- 如果后续实现 Direction B（action correction），A2C2 的 correction head
  设计是最直接的参考
