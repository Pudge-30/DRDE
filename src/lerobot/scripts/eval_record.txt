=== 当前结果 ===

baseline (纯BC, 每10步replan):     94.0%  (推理 ~52 次/episode)
neg_imprv 无replan (ckpt 6000):    62.6%  (推理 11 次/episode)
neg_imprv drift replan=0.10:       77.6%  (推理 ~12.8 次/episode)

drift replan 分析:
- 总 replan 事件 137 次, 48 rollout 中 38 个有 replan
- 70.8% 的 replan 发生在 seg=1（刚生成的 chunk 立刻被判定偏了）
- 存在级联问题：15 条级联链，长度 2-9，平均 4.9

=== 核心问题诊断 ===

CMP 训练链条:
  CMP 训练 → drift 检测能力 → 触发 replan → 重新生成 action chunk
                                                ↑
                                           这一步 = 用同一个模型在当前 obs 重新跑一次

问题在最后一步：replan 做的事情和 baseline 每 10 步做的事情完全一样——从当前
obs 重新生成 chunk。drift replan 77.6% < baseline fixed 94%，说明：
- 检测能力有了（62.6% → 77.6%，replan 确实在起作用）
- 但纠错 ≠ 检测。检测只回答"偏了没有"，而纠错需要回答"怎么回来"

当前 70.8% 的 replan 发生在 seg=1（刚生成的 chunk 立刻被判定偏了），形成级联。
模型从偏离的状态重新规划，生成的 action 依然不好——因为训练数据全是专家轨迹，
模型没见过从偏离状态恢复的情况。

=== 后续方向 ===

方向 A：效率论（推荐，先做）
  目标: success rate 90+（接近 baseline），推理次数远低于 baseline 的 52 次
  故事: CMP 训练 + drift 检测 + 自适应执行 = 接近 baseline 准确率，推理次数降 70%+
  关键: 解决级联问题 + 阈值调优
  阈值选择依据: 可用训练时 l_forward 的均值/分位数作为参考（如 1.5x~2x 均值）

方向 B：将 drift 信号反馈到 action 生成

  B1. 纯推理方案（不改训练）:

    梯度修正法（Embedding-space MPC）:
    ForwardModel 是可微分的：actions → ContentAttention → z2 → FM(z1,z2) → predicted_z3
    执行 k 步后拿到 actual_z3，对剩余 action 做梯度优化：
      remaining_actions.requires_grad_(True)
      z2 = proj_z2(content_attention(remaining_actions))
      z3_pred = forward_model(actual_z3, z2)
      loss = mse(z3_pred, target_z3)
      loss.backward()
      remaining_actions -= lr * remaining_actions.grad
    本质: 嵌入空间 MPC，不改训练，但每次修正需要几步梯度迭代，推理成本增加。

    自适应步长法（更简单，也不改训练）:
    不做二元 replan/不 replan，用 drift 大小决定当前 chunk 还能用多少步：
      drift 小 → 继续执行到 chunk 末尾（省推理）
      drift 中 → 再执行一个 segment 后 replan
      drift 大 → 立即 replan
    直接解决级联问题——不是"偏了就整个扔掉重来"，而是渐进式调整执行长度。

  B2. 需要改训练:
    训练专门的 correction module: action_corrected = action + CorrNet(action, z3_pred - z3_actual)
    问题: 没有 drift 数据（专家数据里不存在"执行偏了"的状态）
    - 训练时加随机扰动制造 drift（扰动分布和真实 drift 可能不一致）
    - 用 RL/online learning 在仿真中采集 drift 数据（工程量大）

=== 务实路线 ===

1. 先跑完 2×2 矩阵，确认 CMP 不伤 BC
2. 方向 A + 自适应步长（不改训练，只改 select_action）
   → 目标: 90+ 成功率 / ~15 次推理
3. 梯度修正法作为 paper 的 future work 或 ablation
